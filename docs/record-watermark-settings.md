# 项目、记录、水印与设置详细设计

> 状态：当前设计
> 适用版本：v0.2.0

## 1. 首页与项目

首页以项目为第一层入口，提供项目搜索、全部记录、总设置和新建项目。搜索只改变当前
列表显示，不修改项目数据。每个项目保存自己的名称、说明和水印参数；拍摄记录通过
项目 ID 与所属项目关联。

新建项目必须通过两层唯一性检查：显示名称忽略首尾空白、连续空白和英文大小写；
安全文件名键还会把文件系统禁止字符统一替换。已有重复项目不迁移，只阻止后续创建
新的冲突项目。

总设置中的水印参数是“新建项目默认值”，只在创建项目时复制，不追溯修改已有项目。
已有项目的水印样式必须进入项目水印设置单独修改。

## 2. 连续拍摄表单

现场记录字段包括：

- 工程部位；
- 工作内容；
- 拍摄人；
- 可选备注。

同一项目完成一张拍摄后，下一张自动保留工程部位、工作内容和拍摄人，备注清空，
避免把上一张的特殊说明误带入新记录。相机取消时保留当前表单内容，不创建可见记录。
系统相机返回后表单不等待水印生成，用户可以立即继续拍摄。

## 3. 记录列表与筛选

项目详情显示本项目记录；首页“全部记录”显示跨项目记录。两处复用同一记录卡片、
缩略图、状态标记和日期筛选组件。

筛选条件采用项目、年、月、日级联规则：

- 项目详情固定当前项目，只显示年/月/日；
- 全部记录增加项目条件；
- 年、月、日控件使用 44dp 高、10dp 圆角的紧凑矩形并排，文字以整个按钮为基准居中；
- 上级条件改变时清除不再有效的下级条件；
- 选项只列出实际存在记录的项目和日期；
- 日期按手机本地时区的 `capturedAt` 计算，缺失时回退 `createdAt`；
- 结果按 `capturedAt ?? createdAt` 倒序。

记录卡片不再直接显示冗长的完整照片编号，而显示`yyyy-MM-dd · 当日序号`；完整编号仍在
详情页中保留。旧编号和新编号都采用相同的短列表标题。

## 4. 预览、详情与编辑

记录卡片根据处理状态选择图片：

- `ready`：显示最终水印图；
- `captured`/`rendering`：优先显示原图并叠加等待或处理中状态；
- `failed`：显示可用图片和错误状态；
- 文件不存在：显示占位图。

详情页显示大图、现场字段、照片编号、拍摄时间、定位结果、文件大小、原图状态和
SHA-256；图片可进入全屏缩放查看。详情操作包括查看原图、删除原图、再次保存水印图、
重新处理和删除整条记录，按钮是否可用由当前状态和文件是否存在决定。

列表进入编辑模式后可以多选记录，并执行：

- 导出所选记录；
- 再次保存所选水印图到相册；
- 清理所选记录的私有原图；
- 删除所选原图、水印文件、相册成片和数据库记录。

全选按钮只选择当前筛选结果中可编辑的记录；再次点击会取消全选。`captured` 和
`rendering` 状态不参与全选。

执行破坏性操作前必须显示清楚影响范围，处理中记录不得进入会与后台任务竞争的编辑或
删除流程。

## 5. 水印内容

水印卡片可显示：

- 项目名称；
- 工程部位；
- 工作内容；
- 拍摄人；
- 拍摄时间；
- 地址、坐标和精度（存在时）；
- 备注（存在时）。

照片编号不显示在水印上，但继续保存在数据库、JPEG 文件名、CSV 和 JSON manifest。
水印语言在拍摄时按当前界面语言写入记录，后台处理即使晚于语言切换，也使用拍摄时的
语言生成中文或英文标签。

## 6. 水印布局和参数

Rust 根据实际文字测量卡片宽度，不使用固定的照片宽度比例作为最终宽度。短内容得到
更窄的卡片，长内容在安全上限内扩展；高度由实际行数、字号、行高和 padding 计算。
布局必须在横拍、竖拍和支持的最低分辨率内避免越界与裁切。

当前可调参数：

| 参数 | 范围或选项 | 生效范围 |
| --- | --- | --- |
| 位置 | 左下、右下 | 项目或新项目默认值 |
| 透明度 | 20%–95% | 项目或新项目默认值 |
| 字体比例 | 80%–160% | 项目或新项目默认值 |
| 强调色 | 绿色、蓝色、橙色 | 项目或新项目默认值 |

不提供自由拖拽、任意字体、任意模板脚本或把编号重新加入水印的开关，以保持不同项目
和不同设备上的输出稳定性。

## 7. 文件命名

照片编号格式为：

```text
{安全化项目名称}-SM-{yyyyMMdd}-{全应用当日序号}
```

项目名称中的空白、控制字符、路径分隔符以及 Windows 禁止文件名字符会替换为下划线；
括号、点号、连字符和 `&` 等非禁止字符可以保留。序号按手机本地日期在所有项目之间
统一递增，从而在不写入项目 UUID 的情况下保持新文件名唯一。最终 `.jpg` 文件名按
UTF-8 计算不超过 255 字节。该规则只作用于新拍照片，旧记录和旧文件不重命名。

## 8. 总设置与关于

总设置包括：

- 外观：跟随系统、浅色、深色；
- 语言：跟随系统、简体中文、English；
- 新项目水印默认位置、透明度、字体比例和强调色；
- SiteMark 应用内数据占用合计及私有原图、私有成片、本地导出、数据库和其他文档明细；
- 存储刷新、拍摄记录管理入口和本地导出 ZIP 清理；
- 关于：应用版本、离线和隐私说明、许可证、第三方声明及 GitHub 代码仓库链接。

存储统计不包含系统相册。清理本地导出前必须二次确认，只删除应用私有 `exports` 目录
中的 ZIP；已分享副本、相册照片、原图和记录不受影响。

“GitHub 代码仓库”显示可识别的仓库地址，点击后交给系统浏览器处理。应用自身不因该
入口获得网络权限。
