# SiteMark 记录列表与存储管理完善设计

## 目标

本轮解决真机使用中确认的四项问题，并补充项目命名约束：记录名称过长、筛选按钮样式不协调、全选不能再次取消、总设置缺少存储占用与清理入口，以及允许创建同名项目。

## 已确认的产品行为

### 新照片名称

升级后新拍照片使用 `{安全化项目名称}-SM-{yyyyMMdd}-{当日序号}`，例如 `云湖之城-SM-20260717-003.jpg`。不再把项目 UUID 写入照片编号或 JPEG 文件名。

序号改为全应用同一天统一递增，避免不同项目的安全文件名发生意外碰撞。旧记录、旧数据库编号和旧相册文件不迁移、不重命名。列表对新旧记录统一显示 `yyyy-MM-dd · 序号`，例如 `2026-07-17 · 003`；详情页继续显示数据库保存的完整编号。

Dart、Android 和 Rust 继续采用同一禁止字符集合与 255 UTF-8 字节上限。新格式仍满足现有 Android 发布和 Rust 归档校验。

### 项目名称唯一性

创建项目前比较已有项目名称：忽略首尾空格、连续空白和英文大小写。同时比较 `safePhotoProjectName` 产生的安全文件名键，防止 `A/B` 与 `A:B` 之类不同输入最终映射为同一文件名前缀。

冲突时不写入数据库，项目表单停留在当前页面并在名称输入框显示明确错误。现有项目不做破坏性迁移；约束只阻止后续新建冲突项目。

### 筛选控件

项目、年、月、日筛选继续在窄屏单行显示，交互和级联规则不变。按钮改为 44dp 高、10dp 圆角的圆角矩形，相邻按钮保留 6dp 间距。文字在整个按钮中水平和垂直居中，下拉箭头固定在右侧，不能再由箭头宽度把文字挤偏。禁用状态使用弱化颜色，但保持相同尺寸和圆角。

### 全选切换

编辑模式中的同一按钮执行“全选/取消全选”切换：当前可选择记录未全部选中则全选；已经全部选中则清空选择。等待处理和处理中记录继续排除。按钮图标和无障碍提示随状态切换，项目记录页与全部记录页复用同一控制器逻辑。

### 存储占用与清理

总设置新增“存储”区块，显示“SiteMark 应用内数据占用（不含系统相册）”以及：

- 私有原图；
- 私有水印图；
- 本地导出文件；
- 数据库及其他应用文档；
- 上述分类合计。

统计服务读取数据库中的原图路径，并扫描应用文档目录中的 `rendered`、`exports`、`sitemark.sqlite` 及其他文件。缺失文件计为 0，路径去重，系统相册中的 `Pictures/SiteMark` 不计入应用私有占用。

存储区块提供刷新、“管理记录”和“清理本地导出文件”。“管理记录”进入全部记录页，由用户按项目/年/月/日筛选后进入编辑模式、全选并清理原图；不在设置页直接批量删除照片。“清理本地导出文件”必须二次确认，只删除应用私有 `exports` 目录中的 ZIP，不影响已经分享到外部的位置、照片、数据库或相册成片。清理后自动刷新统计。

## 技术边界

- 不修改旧照片编号和已发布 MediaStore 名称。
- 不新增数据库 schema；项目唯一性由 `createProject` 事务在写入前统一检查。
- 不新增 Android 权限；存储统计只读取应用自身可访问的文件。
- 不修改筛选条件、记录状态机、水印内容和导出 ZIP 结构。
- 存储统计失败时显示错误与重试按钮，不阻塞主题、语言、定位和关于设置。

## 验收标准

1. 新记录编号不含 `~项目UUID`，旧记录保持原值；同一天跨项目序号不重复。
2. 项目名称规范化后相同或安全文件名键冲突时，新建失败并显示中英文错误。
3. 360dp 宽度下项目/年/月/日筛选保持单行、无溢出，按钮为圆角矩形且文字中心位置不受箭头影响。
4. 连续点击全选按钮依次得到“全选”和“全部取消”，忙碌记录始终不被选择。
5. 设置页能显示四类空间和合计，刷新有效；清理导出文件需确认并在完成后归零。
6. 新增单元和 Widget 测试覆盖编号、项目冲突、列表短标题、筛选几何、选择切换、存储统计与清理；完整 Flutter、Rust、Android 验证保持通过。
