## 下载和安装说明

README 直接链接到 `v0.2.0` 的三个发布资产，并提供 Release 页面入口。明确说明：

- 绝大多数近年手机优先安装体积更小的 arm64 版；
- 不确定处理器架构，或者 arm64 无法安装时，再使用通用版；
- Android 可能要求浏览器或文件管理器获得“允许安装未知应用”权限。

升级说明需要区分：

- 使用同一发布密钥签名的新版本，可以直接覆盖安装并保留应用数据；
- 较早的调试版使用不同证书，Android 可能拒绝覆盖安装。

不能把“先卸载再安装”作为签名冲突的第一条建议。必须先提醒用户导出重要项目和原图。

## 数据安全说明

以下警告必须放在下载或安装说明附近，不能只放在技术章节：

- 项目数据库、项目记录和保留的原图位于应用私有目录；
- 卸载 SiteMark 会清除这些私有数据；
- 已经发布到 `Pictures/SiteMark` 的水印照片通常不会随应用私有数据一起删除；
- 在卸载、更换签名渠道、恢复出厂设置或换机前，应先导出重要项目和原图。

当前版本没有完整的备份恢复和换机迁移功能，因此 README 不能暗示已经具备这些能力。

README 应准确描述已经合并的功能：

- 通过 Android Intent 调用手机系统/厂商相机；

- 缩略图、全屏预览、文件大小和原图保留状态；

- 详情页查看/删除原图以及删除整条记录；
- 主题、中英文、新建项目水印默认值和 GitHub 仓库链接；
- 项目水印字体大小、透明度、位置和强调色；
- 文件名包含项目名称和内部照片编号，但水印画面不再显示编号；
- ZIP 导出水印照片、CSV、JSON manifest 和可选原图。

所有“照片编号显示在水印里”的旧说明必须删除。

## 权限和隐私

权限表必须与发布 APK 一致，包括后台处理依赖带来的通知权限。继续明确说明：

- 位置权限只用于可选的前台定位，拒绝后仍可拍照；
- 不申请 `CAMERA`、`INTERNET`、广泛媒体访问、后台定位和传统存储权限；
- 关于页点击 GitHub 链接时，是外部浏览器负责联网，不代表 SiteMark 本身拥有网络权限。

## 真实性和异常处理原则

- 发布资产链接失效属于阻止 README 发布的问题；
- 版本、权限、测试数量或截图过期时，必须修正或删除；
- 没有真机记录时，不得把某项厂商兼容性描述为已验证；
- SHA-256 只能描述为完整性和追溯数据，不能表述为司法鉴定或第三方存证。
