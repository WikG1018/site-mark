**Goal:** Replace the stale alpha/developer-first README with an accurate Chinese-first download and onboarding page for the signed `v0.2.0` prerelease, backed by current screenshots and a P0/P1-only improvement roadmap.

**Architecture:** Keep the product page in repository-native `README.md` and keep the four current-build screenshots under `docs/images/readme/`. ## Global Constraints

- Work only on branch `agent/readme-v0.2.0-release`; do not write directly to `main`.
- Primary audience: ordinary Android users; contributor and architecture information belongs after user-facing content.
- Current release: signed GitHub prerelease `v0.2.0`, Android 12 / API 31 minimum.
- Recommend the arm64 APK for most phones and the universal APK only as the compatibility fallback.
- State clearly that uninstalling removes the private project database and retained originals; advise export before uninstalling or changing signing channels.
- Do not claim that backup/restore, device migration, or a physical-device compatibility matrix already exists.
- Do not describe the photo number as visible in the watermark; it remains in filenames, records, regeneration, and export.
- Include only the approved P0/P1 roadmap. Do not add the excluded P2 accessibility, large-text, landscape, or extreme-text initiatives.
- Do not retain any screenshot that contradicts the current merged behavior.
- Keep the no-ads, no-account, no-cloud, and no-`INTERNET` product boundary prominent.
