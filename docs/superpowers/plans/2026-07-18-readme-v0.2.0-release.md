# SiteMark v0.2.0 发布版 README 实施计划

> **供执行 Agent 使用：** 必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务执行。每个步骤使用复选框跟踪状态。

**目标：** 把过期的开发者型 README 改为面向普通 Android 用户的中文下载与使用首页，准确介绍已签名的 `v0.2.0` 预发布版，并加入仅包含 P0/P1 的完善路线图。

**实现方式：** 产品首页继续使用仓库原生的 `README.md`，四张当前版本截图继续存放在 `docs/images/readme/`。GitHub Release、已经合并的 `main` 代码、发布 APK 权限和本计划中的重新验证结果是唯一事实来源。所有修改通过独立分支和 Draft PR 发布。

**使用工具：** GitHub Markdown、Flutter、Android API 36 模拟器、ADB、PowerShell 7、Git、GitHub CLI。

## 全局约束

- 只在 `agent/readme-v0.2.0-release` 分支工作，不直接修改 `main`。
- README 第一受众是普通手机用户，技术架构和开发构建放在后半部分。
- 当前发布版是已签名的 GitHub 预发布版 `v0.2.0`，最低 Android 12 / API 31。
- 大多数手机推荐 arm64 APK；只有架构不确定或 arm64 无法安装时才推荐通用版。
- 必须明确提醒：卸载会删除应用私有目录中的项目数据库和保留原图，卸载或切换签名前要先导出。
- 不得暗示当前已经支持完整备份恢复、换机迁移或完整真机兼容性矩阵。
- 照片编号不再显示在水印画面中，但仍用于文件名、记录、重新生成和导出。
- 路线图只允许 P0/P1；用户已经要求排除 P2。
- 不保留任何与当前版本行为矛盾的截图。
- “无广告、无账号、无云端、无 `INTERNET` 权限”必须保持醒目。

## 文件范围

- 修改：`README.md`——下载、安装、数据安全、功能、权限、限制、架构和构建说明。
