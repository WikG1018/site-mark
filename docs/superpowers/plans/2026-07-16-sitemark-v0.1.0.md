# SiteMark v0.1.0-alpha Implementation Plan

> **For agentic workers:** Execute task-by-task with test-driven development and
> verification checkpoints. Do not implement product behavior before observing
> its focused test fail.

**Goal:** Deliver a signed, open-source Android 12+ engineering watermark camera
built with Flutter, Rust, and minimal Kotlin system integration.

**Architecture:** Flutter orchestrates UI and persistence, Pigeon-generated
Kotlin APIs integrate Android system services, and flutter_rust_bridge exposes
file-oriented Rust rendering and export functions.

**Tech Stack:** Flutter stable, Dart, Riverpod, GoRouter, Drift, Pigeon,
flutter_rust_bridge, Kotlin, Rust, Android API 31-36, GitHub Actions.

## Global Constraints

- Repository `WikG1018/site-mark`; app ID `io.github.wikg1018.sitemark`.
- Android minSdk 31, compileSdk/targetSdk 36.
- Chinese and English ship together.
- No ads, analytics, cloud, Google Play Services, or release INTERNET permission.
- System camera only; gallery import is out of scope.
- Originals remain private; published images use MediaStore.
- Apache-2.0 and all bundled font/dependency notices must ship.

## Tasks

1. Bootstrap and pin Flutter/Android/Rust tooling; create the app and verify the
   generated test/build baseline.
2. Define Drift entities, capture state transitions, repositories, and recovery
   behavior through failing Dart tests.
3. Build the bilingual project-first Flutter shell and widget-test navigation,
   forms, record timeline, detail, settings, and export states.
4. Define Pigeon camera/location/media interfaces and implement Kotlin bridges
   with JVM/instrumentation tests, including process-death recovery.
5. Define FRB request/result/error types; implement Rust hashing, orientation,
   watermark layout/rendering, verification, CSV manifest, and ZIP export with
   unit and golden tests.
6. Integrate capture, automatic render/publish, continue-capture, retry,
   regeneration, deletion, and project export with end-to-end fakes.
7. Add bilingual documentation, privacy and contribution policy, dependency
   notices, GitHub CI, signed tag releases, APK checksums, and release checklist.
8. Run all Dart, Rust, Gradle, emulator, APK-manifest, and real-device checks;
   record verified devices and known limitations for `v0.1.0-alpha`.
